# הגדרת Neon — אחסון קבוע להזנות ידניות (סולר / שעות)

מדריך זה מסביר איך להפעיל **שמירה קבועה בענן** להזנות הידניות (סולר ושעות)
באמצעות **Neon Postgres**. בלי זה, הזנות שמבוצעות ב-Streamlit Cloud
עלולות להימחק ב-restart/redeploy (מערכת הקבצים בענן זמנית).

> **חשוב — לפני מיזוג ל-main:** אל תמזג את ה-PR הזה ל-main לפני שביצעת
> את כל השלבים כאן ווידאת ששמירה אמיתית מול Neon עובדת (סעיף 7).
> ראה אזהרה מלאה בסוף המסמך (סעיף 10).

---

## 1. פתיחת פרויקט Neon

1. היכנס ל-https://neon.tech והירשם (אפשר עם חשבון Google/GitHub).
2. לחץ **Create project** (או **New Project**).
3. תן שם לפרויקט, למשל: `m-eylon`.
4. בחר גרסת Postgres (ברירת המחדל מתאימה — Postgres 16/17).
5. אל תלחץ "Create" עדיין — קודם בחר Region (סעיף 2).

> התוכנית החינמית (Free) של Neon מספיקה למערכת הזו, ואינה "נרדמת"
> כמו פתרונות אחרים — לכן נבחרה.

---

## 2. בחירת Region — EU / Frankfurt

ב-מסך יצירת הפרויקט, בשדה **Region** בחר:

```
Europe (Frankfurt)   —   aws-eu-central-1
```

זה ה-region הקרוב והמתאים. אחרי הבחירה לחץ **Create project**.

> ה-Region נקבע פעם אחת ואי אפשר לשנותו אחר כך — ודא שבחרת Frankfurt.

---

## 3. היכן לוקחים את ה-connection string

מיד אחרי יצירת הפרויקט, Neon מציג חלון **Connection Details / Connect**:

1. ודא שנבחר **Connection string** (ולא "Parameters only").
2. ודא שהאפשרות **Pooled connection** מסומנת (מומלץ ל-Streamlit).
3. העתק את המחרוזת המלאה. היא נראית כך:

```
postgresql://<user>:<password>@<host>-pooler.eu-central-1.aws.neon.tech/<db>?sslmode=require
```

> אם סגרת את החלון: לחץ על כפתור **Connect** בראש לוח הבקרה של הפרויקט
> כדי לראות אותו שוב. שמור את המחרוזת במקום בטוח — היא כוללת סיסמה.

---

## 4. הוספת הסוד `NEON_DATABASE_URL`

הקוד קורא את מחרוזת החיבור מתוך משתנה/סוד בשם **`NEON_DATABASE_URL`** בלבד.
**אסור** לכתוב את המחרוזת בתוך הקוד או לדחוף אותה ל-git.

### א. מקומית (על המחשב שלך)

צור קובץ בנתיב הבא (כבר מוגדר ב-`.gitignore`, לא ייכנס ל-git):

```
.streamlit/secrets.toml
```

ותוכנו:

```toml
NEON_DATABASE_URL = "postgresql://<user>:<password>@<host>-pooler.eu-central-1.aws.neon.tech/<db>?sslmode=require"
```

לחלופין, להרצת הסקריפטים מהטרמינל אפשר להגדיר משתנה סביבה:

```powershell
# Windows (PowerShell)
$env:NEON_DATABASE_URL = "postgresql://...sslmode=require"
```

```bash
# Linux / Mac
export NEON_DATABASE_URL="postgresql://...sslmode=require"
```

### ב. ב-Streamlit Cloud

1. היכנס לאפליקציה ב-https://share.streamlit.io
2. **⋮ → Settings → Secrets**.
3. הוסף שורה:

```toml
NEON_DATABASE_URL = "postgresql://<user>:<password>@<host>-pooler.eu-central-1.aws.neon.tech/<db>?sslmode=require"
```

4. שמור — האפליקציה תופעל מחדש אוטומטית.

> כשהסוד מוגדר, מסך "הזנת נתונים ידנית" יציג באנר ירוק:
> **"שמירה קבועה לענן פעילה (Neon)"**.

---

## 5. הרצת בדיקות הקבלה

לפני שסומכים על השמירה, הרץ את בדיקות הקצה-לקצה. הן משתמשות בפרויקט
בדיקה ייעודי (`__neon_test__`) ומנקות אחריהן — לא נוגעות בנתונים אמיתיים:

```bash
python scripts/test_neon_persistence.py
```

הבדיקה מכסה: שמירת סולר, שמירת שעות, שרידות (קריאה בחיבור חדש),
מניעת כפילויות, `replace_month`, בטיחות קלט ריק, ולוג היבוא.

---

## 6. העלאת ההזנות הקיימות ל-Neon (מיגרציה)

אם כבר יש הזנות ידניות מקומיות (תחת `data/manual/`), העלה אותן ל-Neon.
הסקריפט **בטוח להרצה חוזרת** — שורות שכבר קיימות (לפי `row_hash`) מדולגות:

```bash
python scripts/migrate_manual_to_neon.py
```

---

## 7. מה אמור להופיע אם הבדיקה הצליחה

בהרצת `test_neon_persistence.py` המוצלחת תראה:

```
בדיקות קבלה ל-Neon
========================================================
[ok] חיבור ל-Neon תקין.

1+2. שמירת סולר/שעות + קריאה חוזרת
  [ok] שמירת 2 שורות סולר — store_after=2, batch=...
  [ok] נרשם batch_id
  [ok] שמירת שורת שעות

3. שרידות (קריאה חוזרת בחיבור חדש)
  [ok] קריאת 2 שורות סולר חזרה
  [ok] payload נשמר (liters)

4. מניעת כפילויות (שמירה חוזרת של אותן שורות)
  [ok] המאגר נשאר 2 (לא 4)

5. replace_month + בטיחות
  [ok] 05-2026 הוחלף לשורה אחת
  [ok] קלט ריק לא מחק את החודש

6. manual_import_log
  [ok] נרשמו שורות לוג

[ok] ניקוי פרויקט הבדיקה הושלם.
========================================================
סה"כ: 10 עברו · 0 נכשלו.
```

**כל השורות חייבות להיות `[ok]` ו-`0 נכשלו`.** אם יש `[FAIL]` — אל תמזג,
ובדוק את מחרוזת החיבור / ההרשאות.

בנוסף, במסך "הזנת נתונים ידנית" באפליקציה, אחרי שמירה אמיתית תראה:
**"אימות מלא עבר — נשמר לצמיתות בענן (Neon)"** עם מספר השורות שאומתו.

## 8. מה לעשות אם Neon לא זמין

המערכת **לא קורסת** אם Neon לא מוגדר או לא זמין:

- **הסוד לא מוגדר כלל:** המערכת עובדת במצב מקומי בלבד. ב-Streamlit Cloud
  יוצג באנר כתום: **"שמירה קבועה אינה פעילה"** — והנתונים נשמרים זמנית
  בלבד (עלולים להימחק ב-redeploy). הגדר את `NEON_DATABASE_URL` כדי להפעיל.
- **הסוד מוגדר אך אין חיבור** (מחרוזת שגויה / רשת): השמירה לא תאומת,
  המסך יציג שגיאה ברורה ("השמירה לענן נכשלה — אל תסמוך על שמירה זו"),
  והנתונים **לא** יסומנו כ"נשמרו". בדוק:
  1. שהמחרוזת הועתקה במלואה כולל `?sslmode=require`.
  2. שהפרויקט ב-Neon פעיל (לא נמחק).
  3. הרץ שוב `python scripts/test_neon_persistence.py` לאבחון.

---

## 9. מאיפה מגיע כל סוג נתונים

| נתון | מקור | נשמר ב- |
|------|------|---------|
| כרטסת הנהלת חשבונות (תנועות) | `ledger_store` → `master.parquet` | git (נדחף לענן) |
| מאזן בוחן (בקרה) | `balance_store` | מקומי בלבד |
| **הזנות ידניות — סולר** | **Neon** (כש-מוגדר) | **Neon Postgres** |
| **הזנות ידניות — שעות** | **Neon** (כש-מוגדר) | **Neon Postgres** |

- **הכרטסת לא משתנה** — היא ממשיכה להגיע מ-`master.parquet` בדיוק כמו קודם.
- כש-Neon פעיל, הדשבורד ממזג: `master.parquet` (כרטסת) **+** ההזנות
  הידניות החיות מ-Neon (מסומנות `origin='neon_manual'`). הזנות ידניות
  **לא** נאפות ל-`master.parquet` כדי שלא ייספרו פעמיים.
- כש-Neon **לא** מוגדר: ההתנהגות זהה לקודם — ההזנות הידניות נאפות
  ל-`master.parquet`.

---

## 10. אזהרה — אל תמזג ל-main מוקדם מדי

> ⚠️ **אל תמזג את ה-PR ל-main לפני:**
> 1. שהגדרת `NEON_DATABASE_URL` (מקומית **וגם** ב-Streamlit Cloud).
> 2. שהרצת `python scripts/test_neon_persistence.py` וכל הבדיקות עברו
>    (`0 נכשלו`).
> 3. שביצעת הזנה ידנית אמיתית באפליקציה, **ריעננת את הדף**, וראית
>    שהנתונים נשארו — ועדיף גם אחרי redeploy.
>
> רק אחרי שווידאת ששמירה אמיתית מול Neon עובדת — אפשר למזג ל-main.
